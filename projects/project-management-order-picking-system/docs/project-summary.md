# Project Summary

## Context

This project evaluated the implementation schedule for a computerized warehouse order-picking system. The business goal was to reduce manual order-picking labor and automate the movement of goods from storage to shipping.

## Portfolio Framing

The public version highlights the operations analytics value:

- Translating project work into an activity network
- Estimating activity durations using PERT
- Computing schedule timing through CPM forward and backward passes
- Identifying zero-slack activities on the critical path
- Quantifying completion risk with variance and standard deviation
- Recommending project crashing when the target deadline is unlikely

## Interpretation

The expected project duration was 43 weeks, while management requested completion in 40 weeks. The critical path was `B-C-E-F-H-J-K`, and the probability of finishing in 40 weeks was only 10.38%. To reach an 80% probability of finishing by week 40, the expected project duration needed to be reduced to approximately 38 weeks.

The practical recommendation was to crash critical-path activities because shortening noncritical work would not reduce the overall project duration unless it changed the critical path.

## Tooling Note

The original classroom analysis was delivered as a Word report. The portfolio version does not emphasize the report format because the enduring value is the PERT/CPM workflow, schedule-risk analysis, and decision recommendation.
